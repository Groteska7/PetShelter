# PetShelter - Система управления приютом для животных

## Описание проекта

Приложение Windows Forms для ведения учета животных в приюте. Позволяет просматривать, фильтровать, добавлять и удалять питомцев, а также сохранять отчеты в форматах JSON и XML.

## Интерфейс взаимодействия

### Главное окно

![Главное окно](images/main_form.png)

1. **Выпадающий список "Приют"** (ComboBox)
   - Выбор конкретного приюта для фильтрации
   - Опция "(Все приюты)" для отображения питомцев из всех приютов
   - Если не выбрано конкретное значение - учитываются все приюты

2. **Выпадающий список "Вид животного"** (ComboBox)
   - Выбор вида животного: Cat (кошка), Dog (собака), Rabbit (кролик)
   - Опция "(Все виды)" для отображения всех животных
   - Если не выбрано конкретное значение - отображаются все виды

3. **Флажок "Только приюты с открытой территорией"** (CheckBox)
   - При включении отображаются только питомцы из приютов с открытой территорией
   - Важно для животных с клаустрофобией - они могут находиться только в таких приютах

4. **Выпадающий список "Формат отчета"** (ComboBox)
   - Выбор формата сохранения отчетов: JSON или XML
   - При смене формата данные автоматически копируются из файлов старого формата в новый

5. **Кнопка "Показать питомцев"** (Button)
   - Открывает новое окно с таблицей отфильтрованных питомцев
   - Если выбран конкретный приют - доступны функции добавления/удаления питомцев

### Окно с таблицей питомцев

![Окно питомцев](images/pets_form.png)

1. **Информационная панель** (Label)
   - Показывает текущий выбранный приют или "Все приюты"

2. **Таблица DataGridView**
   - Колонки:
     - **Кличка** - имя питомца
     - **Возраст** - возраст в годах
     - **Вес (кг)** - вес в килограммах
     - **Вид** - тип животного (Cat/Dog/Rabbit)
     - **Клаустрофобия** - наличие клаустрофобии (Да/Нет)
     - **Приют** - название приюта, где находится питомец
     - **Доп. информация** - видоспецифичная информация:
       - Для кошек: цвет шерсти
       - Для собак: порода
       - Для кроликов: длина ушей

3. **Кнопка "Добавить питомца"** (Button)
   - Доступна ТОЛЬКО если выбран один конкретный приют
   - Открывает форму для добавления нового питомца
   - При добавлении учитываются ограничения:
     - Животные с клаустрофобией не могут быть помещены в закрытые приюты
     - Превышение вместимости приюта

4. **Кнопка "Удалить питомца"** (Button)
   - Доступна ТОЛЬКО если выбран один конкретный приют
   - Удаляет выбранного в таблице питомца
   - Требует подтверждения операции

5. **Кнопка "Сохранить отчет"** (Button)
   - Сохраняет текущую подборку питомцев в файл
   - Формат файла зависит от выбора в главном окне (JSON/XML)
   - Имя файла: `Подборка_№X_от_YYYY-MM-DD_HH-mm-ss.ext`

6. **Кнопка "Закрыть"** (Button)
   - Закрывает окно с таблицей

### Форма добавления питомца

![Форма добавления](images/add_pet_form.png)

1. **Поле "Кличка"** (TextBox)
   - Ввод имени питомца
   - Обязательное поле

2. **Поле "Возраст"** (NumericUpDown)
   - Ввод возраста в годах (0-50)
   - По умолчанию: 1 год

3. **Поле "Вес"** (NumericUpDown)
   - Ввод веса в килограммах (0.1-200)
   - По умолчанию: 5 кг
   - Точность: 1 знак после запятой

4. **Выпадающий список "Вид"** (ComboBox)
   - Выбор вида животного: Cat, Dog, Rabbit
   - При смене вида динамически меняется блок "Дополнительно"

5. **Флажок "Клаустрофобия"** (CheckBox)
   - Отмечается, если у животного есть клаустрофобия
   - Животные с клаустрофобией не могут быть помещены в закрытые приюты

6. **Блок "Дополнительно"** (GroupBox)
   - Содержимое зависит от выбранного вида:
     - **Для кошек**: поле "Цвет шерсти" (TextBox) и флажок "Стерилизован/а"
     - **Для собак**: поле "Порода" (TextBox) и флажок "Дрессирован"
     - **Для кроликов**: поле "Длина ушей" (NumericUpDown, 1-30 см) и выбор "Тип шерсти" (ComboBox: Short/Long/Rex)

7. **Кнопки "Добавить" и "Отмена"**
   - "Добавить" - создает питомца и добавляет в приют
   - "Отмена" - закрывает форму без сохранения

## Принципы ООП и SOLID в проекте

### ООП Принципы

1. **Инкапсуляция**
   - Все классы имеют private поля и public свойства
   - Конструкторы контролируют создание объектов
   - Методы скрывают внутреннюю реализацию

2. **Наследование**
   - `Cat`, `Dog`, `Rabbit` наследуются от абстрактного класса `Pet`
   - `JsonSerializer` и `XmlSerializer` наследуются от абстрактного `Serializer`
   - Все классы наследуются от базовых типов .NET

3. **Полиморфизм**
   - Абстрактный метод `MakeSound()` переопределен во всех классах животных
   - Метод `ToString()` переопределен для вывода информации о питомце
   - Метод `Equals()` переопределен for сравнения питомцев

4. **Абстракция**
   - Абстрактный класс `Pet` определяет общий интерфейс для всех животных
   - Абстрактный класс `Serializer` определяет общий интерфейс для сериализации
   - Интерфейсы `ICountable`, `IFilter`, `IChangeable` определяют контракты

### SOLID Принципы

1. **Single Responsibility Principle (SRP)**
   - `Pet` и его наследники отвечают только за данные о питомце
   - `Shelter` отвечает за управление коллекцией питомцев
   - `ShelterManager` отвечает за загрузку/сохранение данных
   - `Serializer` и его наследники отвечают только за сериализацию
   - Формы Windows Forms отвечают только за UI

2. **Open/Closed Principle (OCP)**
   - Классы открыты для расширения (можно добавить новые виды животных)
   - Сериализаторы используют обобщенные методы для работы с любыми типами
   - Интерфейсы позволяют добавлять новую функциональность без изменения существующего кода

3. **Liskov Substitution Principle (LSP)**
   - Любой класс, наследующий `Pet`, может использоваться вместо базового класса
   - `JsonSerializer` и `XmlSerializer` могут использоваться вместо `Serializer`
   - Все подклассы корректно реализуют абстрактные методы

4. **Interface Segregation Principle (ISP)**
   - Маленькие, специфичные интерфейсы: `ICountable`, `IFilter`, `IChangeable`
   - Классы реализуют только нужные им интерфейсы
   - Нет "толстых" интерфейсов с ненужными методами

5. **Dependency Inversion Principle (DIP)**
   - Зависимость от абстракций (интерфейсов), а не от конкретных классов
   - `ShelterManager` использует `JsonSerializer` через абстракцию
   - Формы работают с интерфейсами `IChangeable` для добавления/удаления питомцев

## Запуск приложения

### Требования
- .NET 8.0 SDK или новее
- Windows (для Windows Forms)

### Установка и запуск

1. Клонируйте репозиторий:
```bash
git clone <repository-url>
cd PetShelter
```

2. Восстановите зависимости:
```bash
dotnet restore
```

3. Соберите проект:
```bash
dotnet build
```

4. Запустите приложение:
```bash
dotnet run --project PetShelter/PetShelter.csproj
```

### Первый запуск
При первом запуске приложение автоматически создаст:
- 3 приюта с различными параметрами
- 20 питомцев (кошки, собаки, кролики)
- JSON файлы с данными о приютах в папке `Data/`

### Структура данных
Данные о приютах хранятся в JSON файлах в папке `Data/`:
- `Sunrise_Shelter.json`
- `Happy_Paws_Shelter.json`
- `Green_Valley_Shelter.json`

Отчеты сохраняются в папке `Reports/` в выбранном формате (JSON или XML).

## Особенности реализации

1. **Частичные классы (partial)**: 
   - `Pet` разделен на основную часть и часть с конструктором клаустрофобии
   - `Shelter` разделен на основную часть и часть с реализацией `IChangeable`

2. **Ограничения клаустрофобии**:
   - Животные с клаустрофобией по умолчанию: кошки и кролики
   - Животные без клаустрофобии по умолчанию: собаки
   - Животных с клаустрофобией нельзя помещать в закрытые приюты

3. **Сериализация**:
   - JSON использует Newtonsoft.Json с сохранением информации о типах
   - XML использует System.Xml.Serialization
   - Оба формата поддерживают полиморфную сериализацию

4. **Архитектурное разделение**:
   - Бизнес-логика полностью отделена в библиотеке `Model`
   - UI код находится в проекте `PetShelter`
   - Четкое разделение на папки `Core` (классы) и `Data` (сериализация)
